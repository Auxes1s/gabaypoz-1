\begin{abstract}
\noindent
This document is an executable demonstration of the GabayPoz recommender,
version~1.1. It is generated by running \texttt{demo.py} against bundled
questionnaire/program fixtures under \texttt{data/sample/} plus the
v1.1 handoff tables under \texttt{data/processed/team4\_model/}. The script
emits a LaTeX fragment (\texttt{results.tex}) that this wrapper includes
verbatim. Each subsection describes one synthetic student profile, the
top-three program recommendations, the primary school selected for each
program, rank-1 explanation fields, and the three rows shaped for
\texttt{model\_recommendation}. Rebuilding the PDF reproduces this content
from source; see \texttt{Makefile}.
\end{abstract}

\section*{1. About this demonstration}

GabayPoz is a constraint-aware, explainable college program recommender
for Grade~11/12 students in Pozorrubio, Pangasinan. Two rounds of EDA
established that the binding constraints on tertiary access are not
academic preparation but \emph{geography} (mean barangay-to-university road
distance: 76.9~km / 79.2~min) and \emph{household economics} (a 12.7$\times$
education-spending gradient across FIES income deciles). The recommender
honours those findings by scoring programs first, then selecting feasible
schools under each program using barangay-specific travel and affordability
tables.

The pipeline runs in six phases:

\begin{enumerate}\setlength\itemsep{0.15em}
  \item \textbf{Student profile.} Q1--Q9 build a six-dimensional
        affinity vector over (STEM, HEALTH, ARTS, BUSINESS, EDUCATION,
        AGRICULTURE). Internal-interest questions carry 55\% of the
        student score and aptitude questions carry 45\%.
  \item \textbf{Program fit.} Programs are ranked before school
        selection using $0.70$ cosine similarity and $0.30$ normalized
        dot product between the student vector and program affinity
        vector.
  \item \textbf{Market context.} Municipality field saturation is blended
        as a small 10\% context signal. It describes local field presence,
        not guaranteed job demand; missing saturation data falls back to a
        neutral score with a warning.
  \item \textbf{Q12 adjustment.} If the program duration or board-exam
        burden exceeds the student's tolerance, the program score receives
        the documented $0.85$ soft penalty.
  \item \textbf{School selection.} For each candidate program, the
        recommender joins \texttt{university\_program} to
        \texttt{university}, then applies Q11 commute and Q10 affordability
        as hard filters to school suggestions.
  \item \textbf{Output contract.} The recommender returns explanation
        objects for the UI/API and writes exactly three
        \texttt{model\_recommendation}-shaped rows through the caller's
        write callback when persistence is requested.
\end{enumerate}

\noindent
The four profiles below exercise the v1.1 contract: a health-oriented
student tolerant of longer programs (D1), an arts-oriented student with
nearby-only travel (D2), an agriculture-oriented student with moderate
budget and commute tolerance (D3), and a STEM-oriented student with a tight
budget and nearby-only travel (D4).

\section*{3. How to reproduce this PDF}

\begin{verbatim}
cd demo
make            # runs demo.py then pdflatex twice
\end{verbatim}

\noindent
\texttt{make clean} removes the LaTeX intermediates and the generated
\texttt{results.tex}; \texttt{make distclean} additionally removes
\texttt{demo.pdf}. Re-running from scratch reproduces the PDF
deterministically: the recommender is fully stateless, the sample fixtures
and processed v1.1 CSV mirrors are version-controlled, and the ranking rules
are documented in \texttt{docs/tds.md}.
